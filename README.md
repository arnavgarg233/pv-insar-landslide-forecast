# Seven-Day Landslide Forecasting from PCA-Derived InSAR Data with a Random Forest Classifier

[![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Status: accepted](https://img.shields.io/badge/status-accepted-brightgreen.svg)](#publication)
[![Journal](https://img.shields.io/badge/journal-Earth%20Systems%20%26%20Environment-005a87.svg)](https://www.springer.com/journal/41748)

Official code for **seven-day landslide instability forecasting** on the **Palos Verdes Peninsula, California**, integrating a coherence-weighted **Sentinel-1 SBAS** InSAR pipeline, **Principal Component Analysis** of multi-temporal displacement, **NASA GPM IMERG** rainfall, **forecast-oriented pseudo-labeling**, and a **Random Forest** classifier. The framework anticipated ground acceleration during the historic **2024 atmospheric river** events on a held-out chronological test period.

## Headline results

**Test set (held-out, chronologically independent, includes 2024 atmospheric rivers):**

| Metric | Value | 95 % CI (bootstrap, n=36, 10 000 iterations) |
|---|---|---|
| **Accuracy** | **91.67 %** | 80.56 – 100.00 % |
| **Precision** | **95.24 %** | 83.98 – 100.00 % |
| **Recall** | **90.91 %** | 77.77 – 100.00 % |
| **F1-score** | **93.02 %** | 82.35 – 100.00 % |
| **ROC-AUC** | **95.29 %** | 87.00 – 100.00 % |

Discriminative ability is statistically significant against the 50 % random-chance baseline (one-sided bootstrap, **p < 0.001**).

## Highlights

- **PCA isolates physical deformation signals from InSAR time series**, mapping orthogonal modes onto long-term creep (PC1), seasonal shrink–swell (PC2), and transient rainfall-triggered accelerations (PC3).
- **InSAR deformation and rainfall triggers synergistically predict landslide instability** — neither feature stream alone reaches operational performance (see Ablations).
- **Forecast-oriented pseudo-labeling overcomes historical inventory data scarcity** by generating dense, leakage-free labels from future-window cumulative displacement.
- **High-performance 7-day forecasting** with 91.67 % accuracy, 93.02 % F1, and 95.29 % ROC-AUC on a held-out chronological test set covering the 2024 atmospheric rivers.
- **A satellite-driven workflow for regional landslide instability assessment in Palos Verdes**, with a clear pathway to broader adaptation through L-band sensors and additional hydro-meteorological inputs.

## Project overview

Landslide forecasting is constrained by two structural data bottlenecks: **historical inventories are sparse and biased** toward catastrophic events, and **multi-temporal InSAR is high-dimensional and noise-contaminated**. This repository implements a three-stage pipeline that converts these challenges into operationally usable forecasts:

1. **Coherence-weighted SBAS InSAR processing** (MintPy) on a 7-year Sentinel-1 archive (2017 – 2024), producing a high-fidelity LOS displacement time series with explicit ERA5 tropospheric correction.
2. **Unsupervised feature extraction with Principal Component Analysis**, retaining the first three orthogonal modes per orbit (six features, 85.83 % cumulative variance). Each retained component admits a clean physical interpretation supported by quantitative diagnostics:
    - **PC1 — long-term creep**: monotonic strain confined spatially to the Portuguese Bend complex.
    - **PC2 — seasonal shrink–swell**: ~annual periodicity (FFT dominant period **~483 days**), anti-correlated with rainfall (90-day mean, lag 1: **r = 0.15, p = 0.048, N = 179**).
    - **PC3 — transient rainfall-triggered accelerations**: high-frequency pulses temporally coincident with atmospheric river events.
3. **Random Forest classification** of a binary 7-day-ahead instability label, where labels are generated by a **forecast-oriented pseudo-labeling rule** (cumulative future displacement > 3.5 mm threshold, optimized on a chronological validation block, no data leakage).

## Workflow

```
1. Sentinel-1 InSAR (Tracks 71 + 43)  ──▶  SBAS / WLS inversion (MintPy)
                                              │
                                              ▼
2. Displacement time series  ──▶  PCA decomposition (PC1, PC2, PC3 per orbit)
                                              │
3. NASA GPM IMERG rainfall (daily) ──────────►│
                                              ▼
4. Forecast-oriented pseudo-labels (3.5 mm cumulative future displacement)
                                              │
                                              ▼
5. Random Forest classifier (100 trees) ──▶  7-day instability forecast
                                              │
                                              ▼
6. Bootstrap CIs, McNemar tests, ablations, feature importance
```

## Datasets

| Data type | Source / product | Native resolution | Final (after preprocessing) | Temporal resolution |
|---|---|---|---|---|
| **InSAR deformation** | Sentinel-1A/B, C-band, Tracks 71 (asc) + 43 (desc) | ~ — | ~80 m (geocoded) | 6 – 12 days |

**Temporal coverage:** January 2017 – December 2024.
**Interferogram count:** > 5 800 across both tracks.
**Chronological partition:** **60 % train / 20 % validation / 20 % test** (no random shuffling — strict forward-chaining to prevent temporal leakage).

## Random Forest configuration

| Hyperparameter | Value |
|---|---|
| Number of trees (`n_estimators`) | 100 |
| Splitting criterion | Gini |
| Max tree depth | None |
| Min samples per split | 2 |
| Min samples per leaf | 1 |
| Bootstrap sampling | True |
| Features per split | sqrt |
| Random seed | 42 |

Hyperparameters were validated on the chronologically held-out 20 % validation block (no k-fold). The displacement threshold (3.5 mm) was likewise selected on the same validation block to maximize F1.

## Reproduce the paper's results

### Environment

```bash
git clone https://github.com/arnavgarg233/7-day-landslide-forecasting.git
cd 7-day-landslide-forecasting
uv venv && source .venv/bin/activate
uv pip install -r requirements.txt && uv pip install -e .
```

(Conda / pip / Poetry work equivalently — `uv` shown for speed.)

### Main model (RF on PCA + rainfall — paper headline)

```bash
python src/launch.py
# equivalent: python Models/model_b_simple.py
```

### Ablations (Table 5 of the paper)

```bash
python src/launch.py --script Models/model_e_lda.py             # LDA + RF
python src/launch.py --script Models/model_f_pure_rf.py         # raw-feature RF
python src/launch.py --script Models/model_g_fair_rf.py         # fair-comparison setup
python src/launch.py --script Models/model_h_rf_3features.py    # 3-feature RF
```

### Diagnostics and significance tests

```bash
python Scripts/analysis/pc_quantitative_support.py     # PC2 vs rainfall correlation, FFT
python Scripts/validation/bootstrap_confidence.py      # 10k bootstrap test-set CIs
python Scripts/validation/mcnemar_tests.py             # pairwise model significance
python Scripts/benchmark_computational_metrics.py      # timing + memory
```

### Figures

```bash
python Scripts/visualization/generate_roc_curve.py
python Scripts/visualization/combine_existing_figures.py
```

## Limitations and future work

- **C-band decorrelation** in densely vegetated and steep terrain limits broad spatial generalizability. L-band missions (ALOS-2, NISAR) are a clear next step.
- **InSAR temporal sampling (6 – 12 days)** cannot resolve sub-revisit precursory acceleration that initiates and fails between satellite passes.
- **Rainfall resolution** (GPM IMERG Final, 0.1° native) cannot capture sub-kilometer convective events; high-resolution radar nowcasts would improve trigger fidelity.
- **No explicit uncertainty propagation** from WLS InSAR variances through PCA into Random Forest probabilities — full stochastic propagation is left as future work.
- **Site-specific calibration**: the 3.5 mm displacement threshold and PCA loadings are tuned to the Palos Verdes Peninsula. Transfer to other landslide-prone regions requires re-derivation of these parameters and ideally in-situ validation.
- **Operational deployment** would require transition from GPM IMERG-Final to IMERG-NRT and a periodic retraining strategy to handle ground-condition drift.
